Block 33 Workshop: Fitness Trackr

Unit 4

Overview

Complete the following coding challenges using what you know about Node.js runtime, Express.js CRUD routes, middleware, and PostgreSQL databases.

Directions

You've found an awesome open-source project Fitness Trackr that you think can really help you meet some fitness goals in the future. All you've gotta do is practice all of your knowledge and make this awesome API interactive in the browser!

  1. Navigate to the GitHub repo for Block 33: Workshop
  2. Clone the repository and install the dependencies.
  3. Work the tabs below.

There are 7 tabs below.

  • Database and Seed Data
  • GET All Route
  • GET One by ID Route
  • POST Route
  • PUT Route
  • DELETE Route
  • *STRETCH GOAL* - Frontend

Database and Seed Data

Using your knowledge of PostgreSQL and Express.js gained in prior projects, create a database and seed data. You can use past projects as a reference. You will encounter errors, so be sure to troubleshoot them.

Steps to Create Database
  1. Ensure PostgreSQL is running.
  2. Login to PostgreSQL as a super user.
  3. Create a database.
Steps to Initalize Project
  1. Download repository.
  2. Install packages.
Steps to Create Seed Data
  1. Review all scripts in `package.json` file.
  2. Run the script to seed database.
  3. Should receive message: "Finished creating routine_activities!"
Steps to Start Project
  1. Return to review all scripts in `package.json` file, if necessary.
  2. Run the script to start the project.
  3. Should receive message: "Server is listening on PORT: 8080 Get your routine on!"

GET All Route

Check routes that returns all of the data in the database. You can use past projects as a reference. You should not encounter errors at this point. If done properly, then you will be familiarizing yourself with the available set of data.

Steps to Test GET All Route

Using your browser or Postman:

  1. Make a GET request to the "activities" endpoint.
  2. Make a GET request to the "routines" endpoint.
  3. Make a GET request to the "routine_activities" endpoint.

GET One by ID Route

Create a GET route that returns one record from the database by ID. You can use past projects as a reference. You will encounter errors, so be sure to troubleshoot them.

Steps to Test GET One by ID Route

Using your browser or Postman:

  1. Make a GET request to the "activities" endpoint with an ID.
  2. Make a GET request to the "routines" endpoint with an ID.
  3. Uh oh. We've got an error. You will need to debug the "routines" endpoint before continuing.

POST Route

Now you will be adding data to the database by sending a POST request in the body of the request.

Steps for a POST Request
  1. Open Postman or Insomnia.
  2. Make a POST request to the "activities" endpoint. You will need authorization!
  3. Register a new user or login with credentials from the seed database.
  4. Add the token to your POST request and continue adding a new activity.
  5. Make a POST request to the "routines" endpoint. Remember, you will need authorization!
  6. Uh oh. We've got an error. You will need to debug the "routines" endpoint before continuing.
